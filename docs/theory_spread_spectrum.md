# 扩频通信理论速览

## 1. DSSS 基本流程

直接序列扩频把每个数据符号乘以一段高速伪随机码片序列，使信号带宽扩展：

```text
bit -> BPSK symbol -> multiply PN chips -> channel -> correlate PN chips -> bit
```

接收端使用同一 PN 序列相关解扩。期望信号被重新聚合到窄带，未同步的窄带干扰会被摊薄到宽带。

## 2. m 序列

m 序列可由线性反馈移位寄存器 LFSR 产生。只要初始状态非全零，合适抽头可以得到周期为 `2^m - 1` 的最长序列。

本实验约定：

- 寄存器从左到右书写。
- 抽头是从左到右的 1-based 位置。
- 每拍输出右端寄存器。
- 抽头异或得到反馈，反馈插入左端。
- 输出比特映射为双极性码片：0 -> +1，1 -> -1。

## 3. 扩频与解扩

若 PN 长度为 `N`，每个数据符号会扩展成 `N` 个码片：

```text
c[nN + k] = a[n] p[k]
```

解扩时，对每个符号周期做相关：

```text
z[n] = (1/N) sum_k r[nN + k] p[k]
```

若 `z[n] >= 0` 判为 bit 0，否则判为 bit 1。

## 4. 处理增益

扩频因子为 `N` 时，理想处理增益近似为：

```text
G_p = 10 log10(N) dB
```

PN 序列越长，处理增益越高，但同步和带宽开销也越大。
